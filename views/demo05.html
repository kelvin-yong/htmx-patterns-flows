Demo 05

Description
  • Multi-step form
  • Navigate using form buttons: Prev and Next
  • 1 big form, parts hidden by CSS
Technique

General
  • Form replaces itself (same target)
  • Initially the form have empty input values
  • Upon next/prev, the form is rendered with input values as the values being submitted. State is in form, not JS objects
Notes
  • Can combine different approach with history so that user can also navigate with browser's previous and next buttons
