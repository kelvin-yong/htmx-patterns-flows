Demo 04

Description
  • User do something
  • When there is an error, render the error type, display to user
  • The user is shown the error page, with URL
  • Hard reloading the url shows generic error
Technique

General
  • Have a error page route that renders a general error page. This satisfies the hard reloading citeria
  • In other cases, render the page, return response headers 
    "HX-Target":"body"
    and 
    "HX-Push-Url":"someurl"
  • Use response header
    HX-Trigger-After-Settle
    to trigger client events
Notes
  • HX-Replace-Url
    may be better than 
    HX-Push-Url
    in some cases
  • May prefer
    HX-Trigger
    or 
    HX-Trigger-After-Swap
    over 
    HX-Trigger-After-Settle
    if you want the event to be triggered before screen updates are done
